# NeighFund Refactoring

기존 팀 프로젝트 NeighFund를 기반으로  
프론트엔드 구조 개선과 유지보수성 향상을 목표로 리팩토링을 진행한 프로젝트입니다.

---

## 📌 프로젝트 개요

- 지역 공동체 문제 해결을 위한 제안 / 펀딩 / 모임 플랫폼
- 기존 팀 프로젝트를 분석하고 프론트엔드 관점에서 구조 개선 진행

---

## 🛠 기술 스택

- React
- React Router DOM
- CSS
- Fetch API

---

## 🎯 리팩토링 목표

- 로딩 / 에러 처리 개선
- API 호출 구조 정리
- 상태 관리 역할 분리
- 컴포넌트 구조 개선
- 코드 스타일 일관성 확보

---

## 🔧 주요 개선 내용

### 1. 로딩 및 에러 처리 부재 개선
- 사용자에게 데이터 로딩 상태를 명확하게 전달
- API 요청 실패 시 에러 처리 로직 추가

### 2. API 호출 구조 개선
- 분산되어 있던 fetch 호출을 모듈화
- 재사용 가능한 API 구조로 정리

### 3. 상태 관리 구조 개선
- 페이지 단에서 관리하던 상태를 역할별로 분리
- 유지보수성과 가독성 향상

---

## 💡 프로젝트 특징

- 단순 기능 구현이 아닌 기존 코드의 문제점을 분석하고 개선한 프로젝트
- 실제 서비스 흐름을 고려한 프론트엔드 구조 설계 경험
