# NeighFund Refactoring

기존 팀 프로젝트 **NeighFund**를 기반으로  
프론트엔드 구조 개선과 유지보수성 향상을 목표로 리팩토링을 진행한 프로젝트입니다.

---

## 📌 프로젝트 개요

지역 공동체 문제 해결을 위한  
**제안 / 펀딩 / 소모임 플랫폼**입니다.

기존 프로젝트의 프론트엔드 구조를 분석하고  
API 호출, 상태 관리, 컴포넌트 구조를 중심으로 개선 작업을 진행했습니다.

---

## 🛠 기술 스택

- React
- React Router DOM
- React Query
- JavaScript (ES6+)
- CSS

---

## 🎯 리팩토링 목표

- API 호출 구조 일관화
- 로딩 / 에러 처리 개선
- 상태 관리 역할 분리
- 컴포넌트 구조 개선
- 코드 유지보수성 향상

---

## 🔧 주요 개선 내용

### 1. API 호출 구조 개선

- 분산된 fetch 호출을 공통 HTTP 클라이언트로 통합
- API 요청 / 응답 / 에러 처리 로직 일원화
- 인증 처리 흐름 통합 (401 → refresh → 재요청)

---

### 2. 상태 관리 구조 개선

- 서버 상태 / UI 상태 / 폼 상태로 역할 분리
- 서버 상태 → React Query
- UI 상태 → useState
- 폼 상태 → useReducer

👉 상태 책임을 명확히 분리하여 코드 복잡도 감소

---

### 3. 컴포넌트 구조 개선

- 페이지 단위에서 기능 단위 컴포넌트로 분리
- 페이지는 조합 역할만 담당하도록 구조 변경
- 재사용 가능한 UI 구조로 개선

---

### 4. 로딩 및 에러 처리 개선

- API 요청 상태에 따른 로딩 UI 적용
- 에러 처리 로직을 공통화하여 일관된 사용자 경험 제공

---

## 📈 개선 결과

- 약 24개 파일의 API 호출 로직 공통화
- 인증 처리 경로 단일화
- 상태 역할 분리로 코드 가독성 및 유지보수성 향상
- 불필요한 중복 로직 제거

---

- 실제 서비스 흐름을 고려한 구조 설계 경험
- 프론트엔드 관점에서 유지보수 가능한 구조를 고민한 프로젝트
